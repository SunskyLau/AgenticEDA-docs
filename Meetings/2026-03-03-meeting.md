### 2. ✅ 正确的故事线叙事逻辑 (The Right Narrative)

1. **趋势引入 (Trend):** 随着计算成本的降低，**长程运行的Agent工作流 (Long-running Agentic Workflow / Automated EDA)** 正在成为不可忽视的趋势（例如：扔给模型一个数据集，让其跑一整晚进行分析）。这取代了传统的“一问一答 (Turn-by-turn)”模式。
    
2. **痛点揭示 (Gap):** 现有的工具（无论是Chat-based，还是如InsightLens等静态结果展示工具）在面对这种长程、高度自动化的过程时，缺乏 **透明度 (Transparency)** 和 **可控性 (Steerability/Stability)**。用户变成盲目的等待者，无法在分析中途进行干预。
    
3. **我们的方案 (Solution):** 我们提出了一种基于Storyline的交互式可视化界面。它并非改变了EDA的算法底座，而是填补了“长程自动化分析”与“人类探索意图”之间的交互鸿沟，让用户能够理解(Understand)并引航(Steer)分析过程。
    

## 🎨 二、 核心视觉设计 (Visual Design: Storyline)

_(摒弃Ego-centric节点图，全面拥抱Storyline范式)_

### 1. 为什么选择 Storyline?

Ego-centric图（以节点为中心的树状/网状图）缺乏时间深度(Temporal depth)信息，容易导致视觉混乱(Visual Clutter)，且难以表达分析权重的动态变化。Storyline能够天然地编码时间顺序、关系演进以及优先级。

### 2. Storyline 视觉编码重构指北 (Meta-encoding)

你需要打破传统Storyline（线代表人物）的思维定势，赋予其EDA领域的语义：

- **X轴 (横轴):** 代表时间线 (Time) 或 分析深度 (Analysis Depth / Task Sequence)。
    
- **Y轴 (纵轴):** 代表**重要性 (Priority / Importance)** 或 **Agent的工作空间划分**。越靠上的位置表示当前系统或用户认为越重要的分析方向。
    
- **线的语义 (The "Line" - 核心探索点):**
    
    - _方案A (基于数据属性):_ 一条线代表一个 Data Column (例如：Sales, Platform)。线的汇聚代表多个变量产生的联合Insight。
        
    - _方案B (基于语义分类 - 优先尝试):_ 一条线代表一种 Insight Type (如：Correlation, Distribution, Causality) 或者是某个实体的演进（如：特定游戏厂商的分析流）。
        
- **节点/分段 (Segment/Node):** 线上产生的交互节点，代表一组具体的Automated Insights。
    
    - _视觉映射:_ 使用 **大小 (Size)** 或 **透明度 (Opacity)** 来编码该节点的重要性 (Importance Score)。重要节点可以展开显示缩略图或具体Insight。
        

### 3. 可扩展性应对 (Scalability)

- 不要过度焦虑节点爆炸问题。长程分析的初期和中期，Storyline足以承载。
    
- **解决方案:** 引入 **迷你地图 (Mini-map)** 用于全局导航(Navigate)，结合缩放(Zooming)解决时间轴过长的问题。
    

## 🕹️ 三、 交互与控制设计 (Interaction & Steering)

_(界面的灵魂：如何让用户干涉Agent)_

交互设计的目标是：如何将用户的探索策略 (Steering intent) 转化为Agent的分析方向。

- **分支 (Branching):** 在某个节点引发新的思考，手动添加一个分支（Prompt介入），让Agent向新方向探索。
    
- **提权/置顶 (Prioritizing - "拉直操作"):** 拖拽 (Drag) 某条线/某个Insight到Y轴最上方，或者将其“拉直”，系统据此增加该方向在模型层的计算权重 (Thinking intensity)。
    
- **融合 (Merge):** 强制要求Agent将不相关的两条线（变量/任务）结合起来寻找Insight。
    
- **终止 (Terminate/Delete):** 对于无意义的探索分支，用户可以直接掐断，线变灰，Agent停止该方向的算力消耗。

## 🧮 四、 算法与系统支撑 (Metrics & System)

_(由Miaoyan主导，用于支撑视觉呈现)_

**Importance Score (重要性/趣味性计算公式):** 为了决定哪些节点放大、哪些置顶，需要一个合理的复合评价指标。

- `Score = w1 * 统计学特征 (Statistical metrics) + w2 * 大模型打分 (LLM/Agent Scoring) + w3 * 数据集覆盖率 (Data Coverage / Surprisingness)`
    
- _教授批注:_ 指标不需要完美，但必须**“言之有理” (Justifiable)**。确保各个维度的定义正交（没有太多重叠），能够自圆其说即可应对Reviewer的Challenge。
