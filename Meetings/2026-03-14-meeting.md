# 交互式分析 Agent 协同系统：开发与写作推进指南 (Master Document)

**文档目标:** 提炼刘阳天与缪延会议中的核心共识，修复原始转写中的语义偏差，为当前论文的理论写作（交互意图分类）和系统开发（前端UI/UX、后端架构、Agent通信机制）提供绝对清晰、结构化、可直接执行的行动指南。

## 核心理论框架：基于意图的大模型引导机制 (Steering Intents)

_【论文写作重点】本节主要用于构建论文的 Interaction Space / Steering Taxonomy 章节。我们最终确立了 5+1 的交互框架，放弃了过于细碎的底层交互，转而从用户高层意图出发。_

### A. 软引导 (Soft Steering - 基于 Prompt 注入)

这类操作本质上是对大模型上下文和指令的修改，**不涉及强行中断底层代码执行**。

1. **Focus (聚焦/深挖)**:
    
    - **定义**: 针对当前 storyline 上的某个对象（Insight/Block），让 Agent 基于此节点继续 Drill-down（深入探索）。
        
    - **实现**: 将选中的 Block 转化为上下文发送给 LLM。
        
2. **Surprise (压制/避开)**:
    
    - **定义**: 告诉 Agent “少做或别做”某个方向的探索，属于一种负向反馈（Negative constraint）。
        
3. **Expand (发散/扩展)**: _(语音识别误作 spend)_
    
    - **定义**: 在现有节点基础上适度联想，拓宽广度（elaborate）。与 Focus（重深度）形成互补。
        
4. **Reframe (重构/视角转换)**: _(语音识别误作 refrain)_
    
    - **定义**: 改变分析角度，或者关联/合并（Merge）几个现有的节点产生新的分析视角。
        
5. **Inform (外部知识注入)**: _(语音识别误作 infor m/The form)_
    
    - **定义**: 作为一个全局的约束指令或外部上下文，注入给整个分析过程。
        

### B. 硬控制 (Hard Control - 基于底层机制干预)

这类操作属于细粒度控制，**直接干预 Agent 的执行流**。

- **强制接受/否决 (Accept/Reject)**: 对于自动生成的分析计划进行裁决。
    
- **强制暂停/终止 (Pause/Terminate)**: 强行中断正在运行的 Analyzer（子Agent）。
    
- **强制插入计划 (Add Block)**: 用户手动新建一个 Block，强制 Master Agent 执行该特定计划。
    

**💡 核心 Insight (来自会议讨论):**

- **极简主义原则**: Coding Agents (如 Cursor) 证明了用户更倾向于简单的交互。不要设计过多的菜单和标记。如果用户当前的 Steering 偏离了初始 Goal，**让 Agent 自己去判断是否放弃原 Goal**，而不是在 UI 上弹出复杂的“是否放弃原任务”的选项。
    
- **合并冗余**: Focus 和 Expand 在操作层面有一定重合。后续开发中可以先整合为一个通用的 "Contextual Steer"，内部再细化 Prompt。
    

## 前端交互与视觉设计优化 (UI/UX & Visual Encoding)

_【前端开发清单】整理了所有 UI 细节、视图联动逻辑以及布局修复方案。_

### 1. 视图联动与过滤机制 (View Coordination & Filtering)

- **单点排他与互相取代**:
    
    - 如果在 Storyline 中点击了某个 Filter（如某个 Publisher），再去点击某个具体的 Block，**两者应当互相清空（取代）**。即：专注于当前点击的实体，不要让过滤条件无限叠加导致视觉混乱。
        
    - **例外保留**: 图例（Legend）点击与 Storyline 内部高亮保持联动。
        
- **单双击逻辑区分**:
    
    - **单击 (Single Click)**: 单选/切换（Switch/Select one）。
        
    - **双击 (Double Click)**: 多选/固定（Multi-select/Pin）。
        
- **Coverage Map 逻辑**: 在右侧 Coverage Map 中的点击操作保持为**取并集 (Union)** 逻辑。
    

### 2. UI 元素精简与布局重构

- **Inspector / Legend 区域瘦身**:
    
    - **删除**: 去掉 "AVG" 和 "Legend" 字样（没有必要，用户不在意）。
        
    - **重排**: 只保留 `Count` 和 `Importance` (例如 `0.00` 到 `0.56`)，并将它们分别置于左右两侧。
        
    - **顶部计数器**: 将“当前选中/总共多少个 Insight (如 23 insights)” 放在该区域的最右侧顶部。
        
    - **高度统一**: 移除不必要的空白，将该条目的高度压缩，与主界面其他元素保持统一。
        
- **全局 Inform 入口设计**:
    
    - 参考主流大模型产品，在主对话框（Chat/发送按钮）左侧对称放置一个 **`+` 加号按钮**。
        
    - 点击加号后，弹出一个**可伸缩的悬浮面板/向左侧展开的弹窗**，用于管理（添加、删除）注入的外部知识或约束。
        

### 3. 视觉编码修正：节点重叠与字号问题 (Visual Overlap Bug)

- **当前 Bug**: 高亮状态或生成结果多时，Block 节点（尤其是顶部的节点）会挤压、重叠（粘在一起）。
    
- **底层原因分析 (缪延提供)**:
    
    - 字号大小映射逻辑：当前基于左右两块 Block 的平均 Importance 计算字号，这部分逻辑保留。
        
    - 布局约束冲突：节点之间存在最小间距 $d_1$ (相关线) 和 $d_2$ (不相关线)。当缩放比例变化或节点过多时，纵向空间不足，导致顶部空间被强行挤占。
        
- **Actionable 修复方案**:
    
    - **调小全局** $d_2$ **值**（不相关线的间距）。因为设定最多同时展示 5-6 个 Agent，只要 $d_2$ 足够小，就不会出现顶部重叠卡死的问题。
        

## 后端架构与 Agent 控制逻辑 (Backend Architecture)

_【后端开发与架构设计】这部分涉及 Master Agent 与 Analyzer 的通信协议修改。_

### 1. 异步运行状态的视觉占位 (Asynchronous Rendering)

- **需求**: 当自动模式下有 3-4 个 Agent 正在跑（但还没跑完）时，前端需要优先显示这些 Block。
    
- **实现**: 前端根据后端状态，直接显示**带有占位符的空框架 (Skeleton/Empty Frame)**。不需要等待全部内容生成完毕。
    

### 2. Steering 动作的上下文转化 (Prompt Engineering)

- **Soft Steering 实现路径**:
    
    - 用户在 UI 上右键某 Block 执行 `Focus`, `Surprise` 或 `Reframe`。
        
    - 前端将该 Block 的 ID/内容作为 Context 提取，结合用户意图，等价于**在对话框中输入了一段定制化的 Prompt** 发送给后端。
        
- **Inform 实现路径**: 直接在 System Prompt 或 Global Context 中动态添加该约束。
    

### 3. 复杂控制流的管理 (Timeline / State Management) - [难点/后续阶段]

- 当执行**硬控制 (Hard Control)**，如“取消某个正在运行的 Plan”或“手动新建一个 Block”时，会破坏大模型原有的执行上下文。
    
- **系统改造要求**:
    
    - **Timeline 记录**: 后端必须维护一个针对 Prompt 的 Timeline。如果用户取消了 Plan #3，系统要在 Timeline 中记录“用户终止了 Plan #3”。
        
    - **状态同步回传**: 新增或取消的操作必须显式地 (explicitly) 告知 **Master Agent**。例如，告诉它“用户在此批次中人工创建了一个新计划，特征如下…”，否则下一次生成时 Master Agent 会产生逻辑断层（幻觉）。
        

## 优先级与下一步行动计划 (Action Items)

**🔴 高优先级 (Phase 1: 易实现且对论文截图展示最关键)**

1. **[前端] UI 清理**: 完成 Inspector 区域的瘦身，移除冗余文字，调整布局高度。
    
2. **[前端] 视图联动修复**: 实现 Filter 与 Block 点击的“互相取代”逻辑，实现单双击逻辑区分。
    
3. **[前后端联调] Soft Steering 闭环**: 实现 `Focus`, `Surprise`, `Expand`, `Reframe`。UI 上通过右键菜单（或拖拽到输入框）实现，后端将其转化为等价的 Prompt 发送。
    
4. **[前端] Inform 弹窗**: 在输入框左侧添加 `+` 号，实现外部知识管理的交互面板。
    

**🟡 中优先级 (Phase 2: 完善视觉与异步体验)**

1. **[前端/算法] 布局算法优化**: 调小约束距离 $d_2$，彻底修复 Block 节点在高密度下的重叠问题。
    
2. **[前端/后端] 占位符渲染**: 前端读取后端 Agent 运行状态，为未完成的计算任务渲染“空框架”。
    

**🟢 低优先级 (Phase 3: 复杂的底层架构修改 - 建议开题报告后推进)**

1. **[后端] Timeline 与强控制**: 实现暂停、终止某个特定子 Agent 的功能，并确保该状态能准确反馈给 Master Agent 的上下文。
