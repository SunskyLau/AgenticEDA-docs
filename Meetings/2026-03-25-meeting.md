## 模块一：系统工程与交互优化 (System & Interaction Refinement)

_注：系统整体 Ready，仅需根据讨论进行少量但关键的逻辑调整。_

### 1. Agent 调度与即时响应逻辑优化 (🔥 **Critical**)

- **当前问题:** 用户在 Agent 运行计划的过程中输入新的指令（如 Focus 特定列，或 Create 新计划），系统可能会等待当前批次跑完才响应，存在滞后感。
    
- **Weng's Directive (如何做):** * 实现**即时响应（Immediate Response）**。当用户在当前 Session 创建新 Plan 或施加 Focus 约束时，必须在当前 Session 立刻生效，保持前后端状态一致。
    
    - **赋予 Planner 动态调度能力:** 让主控 Agent (Planner) 评估用户的新指令。如果新指令（如只关注某一列）与当前正在执行的子线程计划相冲突或无关，Planner 应自动对无用计划执行 **Pause（暂停）** 或 **Terminate（终止）** 操作，并立即生成/修改符合新约束的 Plan。
        
- **Why:** 提升系统的 Steering (引导) 能力，体现 Human-in-the-loop 的即时反馈价值。
    

### 2. UI/UX 视觉优化

- **任务:** 为论文中提到的每一种 Intention Level / Execution Level 功能操作（如 Focus, Ignore, Elaborate 等）生成并添加对应的 Icon。
    
- **工具建议:** 可以使用类似 Midjourney/DALL-E 等 AI 图像生成工具快速产出风格统一的 Icon，提升系统界面的完善度和论文插图的质感。
    

## 模块二：案例研究设计与执行 (Case Study Strategy)

_注：不要让专家自由发散，必须有策略地引导他们展示系统的不同核心能力。_

### 1. Case A: 体育数据集 (Sports Data) —— 侧重于 “Understanding & Steering”

- **数据特点:** 规模大、列数多、行数多（几万条），专家初始状态通常是“毫无头绪”。
    
- **展示核心:** 展示系统在**探索性数据分析（EDA）**过程中的广度探索与动态引导能力。
    
- **执行剧本 (Playbook):**
    
    1. 让专家面对庞大数据，先让 Agent 自由生成初步计划。
        
    2. 观察 Agent 抛弃/未使用的某些数据列。
        
    3. 引导专家发现某些被遗漏的列其实很有趣，利用系统的 **Focus** 或 **Create** 功能，强制 Agent 关注这些列。
        
    4. 展示系统如何响应用户的 Steering，修改方向并深挖 insight。
        

### 2. Case B: 社媒数据集 (Social Media Data) —— 侧重于 “Directional Analysis (定向分析)”

- **数据特点:** 专家可能已经有了一定的先验知识或特定的分析目标。
    
- **展示核心:** 展示系统在强约束条件下的执行力、以及挖掘深层模式（Interesting Patterns）的能力。
    
- **执行剧本 (Playbook):**
    
    1. 一开始就给出相对明确的 Constraints（例如：只看某两个 Column 之间的关联）。
        
    2. 跳过大范围的 Steering，直接进入深度的 Understanding 阶段。
        
    3. 展示系统如何在一两个特定的视角下，帮助专家快速理清逻辑并得出最终结论。
        

### 3. 案例记录规范 (Recording Protocol)

- **操作:** 使用屏幕录制（如钉钉录屏、OBS）完整记录专家的操作过程，因为大模型具有不可重复性（Non-deterministic）。
    
- **关键干预技巧:** 在录制过程中，**主动提醒并引导专家使用 Pause（暂停）功能**。不要让 Agent 跑得太快导致专家错过信息。这不仅是工程妥协，更是学术发现（见模块四的理论包装）。

### 3. 核心学术贡献包装 (🔥 Weng's "Golden Nugget" for Discussion)

- **关于“Pause (暂停)”功能的学术升华:**
    
    - **不要只把它当成一个工程 Bug fix！** 翁老师指出，这是一个非常好的 HCI 洞察 (Insight)。
        
    - **论文写作角度:** 当 AI 代理自动化进行高密度的探索时，会导致专家产生**“信息过载 (Information Overload)”**。专家需要（甚至渴望）一个 Pause 功能来中断自动化的 EDA 流程，以便他们有时间去消化、评估现有的 Insights，然后再决定下一步的 Steering 策略。
        
    - **结论:** 这种“暂停-思考-再干预”的模式，深刻体现了 Human-AI Collaboration 在数据分析中的真实需求。必须将其作为一个重点写入论文的 Discussion 部分！
