## 1 当前可视化设计的问题

- **信息量不足**：现有 ego-centric layout 难以让用户直观理解分析内容与语义
- **数据语义缺失**：过度强调 insight type，忽略数据语义——用户无法判断"正在分析什么数据"
- **视觉混乱**：节点内含多个 atomic insight，导致连线交错；缺乏路径语义解释（如"为何从 A 到 B"）
- **布局干扰**：coverage map 与 structure 视图相互干扰，缺乏嵌入式（embedded）与分层聚焦交互能力

---

## 2 可视化改进方向

### 2.1 Storyline 类布局
- X 轴：从左到右按分析进程分布
- Y 轴：按数据相似性/关联性排布，同类数据流聚集成束并标注

### 2.2 Coverage Map 提升为主视图
- 以数据列为轴，insight 自由分布在对应数据线周围（非严格网格）
- 支持按列筛选与模式探索

### 2.3 数据语义优先编码
- 主视图中先呈现关键数据组合（如 sales-volume-titles 关系）
- 再编码分析目标（如 compare）与 insight type

### 2.4 文本注解与关键词标注
- 在边（task）和节点（summary）上添加短语/主题标签，提供即时语义概览

## 3 交互与监控机制需求

| 需求 | 说明 |
|------|------|
| Step-wise 探索 | 允许用户在任意节点暂停、干预、调整后续分析方向（剪枝、指定优先分支） |
| 路径级解释 | 每条分析链清晰展示承前启后逻辑（如"因上步未定位 high-value titles，故本步深入 identify"） |
| 聚焦式交互 | 点击 branch 高亮完整路径，隐藏其他分支，降低认知负荷 |
| 多粒度查看 | overview 层显示语义关键词；detail 层提供图表 + 文本 + 推理依据的综合呈现 |

## 4 Agent 驱动 EDA 范式的合理性论证

- **范式演进**：Agent 探索是面向未来的范式（类比 Google Search → LLM 检索），适用于需深度语义理解的复杂分析场景
- **对比规则启发式**：相较 CoInsight、Reactable 等，Agent 能处理特殊数据语义与开放性任务，但需解决计算成本与可控性平衡
- **用户天然的 step-wise 需求**：初始意图常模糊，需通过渐进式探索澄清目标，而非依赖一次性全自动报告（易致"wall of text"体验）

## 7 要点回顾

1. **可视化需呈现数据探索进度**：当前设计未能有效传达探索状态，需优化以增强用户控制能力，实现逐步干预
2. **Agent EDA 框架贡献定位**：需通过技术评估证明优于现有 LM 驱动方法，强调用户干预能力的独特价值，避免与全自动化工具直接竞争
3. **交互式分析的必要性**：用户初始目标不明确，逐步引导分析比一次性自动化更符合实际需求
4. **Storyline 布局优化**：通过主题关联与路径标注增强数据语义表达，探索 section layout / circular layout 等灵活结构
5. **Agent 驱动分析的未来趋势**：随着模型性能提升，基于 agent 的分析将成为主流，需前瞻性设计
